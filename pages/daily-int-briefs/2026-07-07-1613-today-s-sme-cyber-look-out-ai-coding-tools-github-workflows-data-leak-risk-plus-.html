Today’s SME cyber look-out: AI coding tools & GitHub workflows data leak risk, plus router backdoor warnings

What small and medium-sized businesses should look out for today.

Moderate Tuesday 07 July 2026, 16:13 UK time
Today’s look-out: Supplier/SaaS AI workflow data leakage + small office router exposure

What to look out for today

  • GitHub “agentic”/AI-driven workflows may leak private code or secrets if they’re allowed to read across repositories and can be influenced by public issues.
  • Small office / home office router risk: a CERT/CC warning describes an undocumented admin backdoor in some Tenda router firmware.
  • Microsoft Windows backup behaviour change: Windows settings backup/restore is expected to be enabled by default on some organisation-managed Windows 11 devices after upgrade, which may affect what gets stored in the cloud and how machines are restored.

Why this matters to smaller businesses

SMEs increasingly rely on SaaS and “helpful” automation (AI assistants, build pipelines, scripted workflows) because they reduce admin time. The downside is that a single misconfigured permission can expose a lot of information quickly (source code, internal docs, configuration, tokens, customer data).

Separately, many small organisations still use consumer-grade routers in offices, pop-up locations and staff homes. If a router model has a hidden admin access method, it becomes an easy path to intercept traffic, change DNS, or gain a foothold for further fraud.

Finally, default backup/restore changes in Microsoft-managed devices are usually positive for resilience, but they can create governance questions (what settings are backed up, where, who can restore, and what happens during offboarding).

Warning signs

  • Unexpected access patterns in GitHub: sudden spikes in workflow runs, unusual read activity across multiple repos, or “helpful” bots posting verbose summaries that include internal file contents.
  • Secrets showing up where they shouldn’t: API keys, tokens, connection strings, or customer identifiers appearing in build logs, issue comments, or automated “analysis” outputs.
  • Router/redirection symptoms: staff reporting “certificate warnings”, being redirected to odd login pages, or multiple users suddenly failing to reach known services (banking, M365, payroll) unless on mobile data.
  • Windows device surprises after upgrade: users seeing new backup prompts, restored settings they didn’t expect, or confusion about whether settings follow them to new machines.

How attackers may exploit the situation

  • Workflow prompt/issue manipulation: an attacker opens a normal-looking public GitHub issue designed to influence an AI-enabled workflow/agent into fetching or revealing private repo content (or summarising sensitive files).
  • Permission overreach: if the workflow/agent has broad read access, the attacker doesn’t need to compromise an employee account to cause damage—misconfiguration does the heavy lifting.
  • Router foothold for fraud: a compromised router can be used to reroute traffic, capture credentials, or facilitate Business Email Compromise-style payment diversion by spying on comms and logins.
  • Backup/restore misuse: if identity controls and offboarding are weak, backup/restore features can unintentionally preserve settings and access paths longer than intended.

What to do today

  • If you use GitHub automation/AI agents: confirm what repositories they can read, what they can post back publicly, and whether public issues can trigger actions that access private data.
  • Scan for exposed secrets: check recent workflow logs and automated comments for accidental inclusion of tokens, passwords, connection strings, or customer data. Rotate anything suspicious.
  • Router hygiene: identify your office routers (make/model/firmware) and confirm who is responsible for updates, admin access and configuration backups—especially for satellite offices and home workers.
  • Microsoft backup governance: ask your IT support what Windows settings will be backed up by default on Entra-joined devices, where it’s stored, and how restores are controlled.

Ask your IT provider

  • Do any of our GitHub repos use agentic/AI workflows? If yes, which identities do they run as and what repo scope do they have?
  • Can a public GitHub issue trigger automation that reads private repos or posts outputs publicly (issues, PR comments, logs)? What guardrails are in place?
  • Do we have an inventory of office and remote-worker routers, and a process for urgent vendor advisories (including consumer devices)?
  • After Windows 11 upgrades, will settings backup turn on by default for us? What’s the policy for backup/restore, leavers, and shared machines?

Patch watch - only one short paragraph, and only if relevant

CERT/CC has warned about an undocumented admin access backdoor affecting certain Tenda router firmware, and researchers have described a GitHub workflow data-leak scenario tied to agentic workflows. For SMEs, the priority is not “chasing CVEs” but confirming (1) whether you use affected router models and (2) whether your GitHub automation/AI agents have overly broad permissions or can be influenced by public inputs.

One action today

Audit any GitHub agentic/AI workflows today: ensure they cannot be triggered by public issues to access private repos, and reduce their repo permissions to the minimum needed.
