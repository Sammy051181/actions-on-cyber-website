Today’s SMB cyber lookout: Microsoft 365 “passkey enrolment” vishing + urgent web app exposure checks

What small and medium-sized businesses should look out for today.

High Wednesday 08 July 2026, 18:51 UK time
Today’s look-out: Voice phishing (vishing) to hijack Microsoft 365 logins, plus web app compromise leading to ransomware/disruption

What to look out for today

1) Microsoft 365 voice scams (“vishing”) prompting passkey enrolment. Reports describe attackers calling staff and posing as IT/security, asking them to enrol a new Microsoft Entra passkey for “security reasons”. The goal is typically account takeover.

2) Web app exposure risk from an actively exploited ColdFusion issue. Even if you don’t think you run ColdFusion, it can exist in older internal apps, supplier-hosted portals, or forgotten servers.

Why this matters to smaller businesses

  • One compromised Microsoft 365 account can lead to invoice fraud, payroll diversion, mailbox rule tampering, and customer data exposure.
  • Vishing works on busy teams: finance, reception, HR and new starters are often targeted because they’re helpful and process-driven.
  • Web app compromise is a common ransomware entry point, especially where legacy apps sit outside modern monitoring.

Warning signs

  • A call claiming to be “Microsoft”, your “IT support”, or an “MFA team” asking you to set up a passkey, approve a sign-in, or read out a code.
  • Pressure tactics: “You’ll be locked out in 10 minutes” or “We’re seeing suspicious activity right now.”
  • Unexpected Microsoft 365 prompts or new authentication set-up screens that you didn’t initiate.
  • Reports from staff that they were asked to install a remote support tool or visit a link during a call.
  • Unexplained website/app instability, spikes in server load, or sudden security alerts related to public-facing web apps.

How attackers may exploit the situation

  • Social engineering + identity changes: The attacker persuades a user to enrol a new sign-in method, or to approve an action that hands over access.
  • Mailbox takeover: Once in, criminals search for invoices, change bank details in ongoing email threads, or set hidden forwarding/inbox rules.
  • Follow-on compromise: A single account can be used to phish customers/suppliers from a trusted address, or to access connected SaaS tools.
  • Public-facing app entry: If a web app platform is actively exploited in the wild, attackers may scan for exposed instances, gain a foothold, then pivot to file servers and backups.

What to do today

  • Brief staff (2 minutes): “We will never call you to set up a passkey or ask for codes. If you get such a call, hang up and report it.”
  • Set a simple rule: Any request to change login methods, bank details, supplier payment details, or payroll details must be verified via a known-good number (not the one provided on the call/email).
  • Check Microsoft 365 basics: confirm MFA is enabled for all users, review any recent changes to authentication methods, and look for unusual sign-ins.
  • Reduce blast radius: ensure shared mailboxes and finance mailboxes have the strongest protection and least privilege necessary.
  • Inventory internet-facing apps: confirm what is exposed publicly (including any legacy app servers) and who owns patching/monitoring.

Ask your IT provider

  • Are we seeing any signs of vishing-driven account takeover attempts (unusual sign-ins, impossible travel, repeated MFA prompts)?
  • Can you confirm who can enrol new authentication methods and how we detect/alert on suspicious enrolments?
  • Do we have alerts for mailbox forwarding rules, newly created inbox rules, and changes to payment-related mailboxes?
  • Do we run (or inherit via a supplier) any ColdFusion-based applications, portals, or legacy servers? Who is responsible for patching and how quickly?
  • If a web app is compromised, what’s our containment plan (isolation, credential resets, restore process) and how fast can we execute it?

Patch watch - only one short paragraph, and only if relevant

CISA has highlighted an actively exploited Adobe ColdFusion issue and set a rapid remediation deadline for US federal agencies. For SMEs, the key action is to confirm whether any ColdFusion servers exist in your environment (including older apps) and ensure your IT provider has a process to rapidly address actively exploited internet-facing software—especially anything hosting customer or staff-facing portals.

One action today

Send a same-day staff message: “If anyone calls asking you to enrol a new Microsoft 365/Entra passkey or share codes, hang up and report it—IT will never ask.”
