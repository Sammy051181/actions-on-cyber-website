Today’s SMB cyber lookout: WhatsApp “business document” messages leading to remote access

What small and medium-sized businesses should look out for today.

High Tuesday 23 June 2026, 11:43 UK time
Today’s look-out: Messaging-app phishing leading to remote access and account compromise

What to look out for today

Watch for WhatsApp messages (especially via WhatsApp Desktop/Web) that pretend to share business documents (invoices, purchase orders, shipping notes, scanned PDFs) but actually try to get staff to open a script file. Reports indicate this can lead to remote access on the PC, including the installation of legitimate remote monitoring/management (RMM) tooling that attackers then abuse.

Separately, there is reporting on credential theft from compromised FortiGate firewalls, which can lead to remote access into business networks if a supplier or your own perimeter device is affected.

Why this matters to smaller businesses

  • WhatsApp is a business channel for many SMEs (customers, suppliers, trades, schools, charities). It’s easy for a convincing “document” message to reach finance/admin staff.
  • Remote access is the fast path to disruption: once attackers get a foothold they may steal data, tamper with payments, or use the access for ransomware later.
  • Abused “legit” tools reduce suspicion: if attackers install real RMM software, it may look like normal IT activity unless you’re checking.
  • Network edge compromise has ripple effects: if firewall credentials are stolen, attackers may access email, file shares, and cloud services through trusted connections.

Warning signs

  • WhatsApp message urging urgency: “please review today”, “overdue”, “final notice”, “payment confirmation”, “delivery issue”.
  • Unexpected attachments or links, especially if the “document” is not a normal PDF/Office file.
  • Prompts to open a file that looks like a script (e.g., odd file extensions) or anything that triggers security warnings.
  • New remote-control prompts, new “support” pop-ups, or the mouse moving on its own.
  • New admin tools appearing on a PC, or a device suddenly showing up in your IT/RMM portal that no one can explain.
  • Unusual VPN/firewall logins or logins from new locations (if you or your IT provider monitor these).

How attackers may exploit the situation

  • Document pretext: attackers pose as a supplier/customer and send a “business doc” via WhatsApp.
  • Execution for access: opening the file can give the attacker remote access, sometimes by installing legitimate RMM software to blend in.
  • Credential theft and lateral movement: once on a PC, attackers may grab saved passwords, hijack email, and target finance processes (invoice fraud) or deploy ransomware later.
  • Perimeter credential theft: stolen firewall credentials can be used to access internal systems and cloud apps via trusted routes.

What to do today

  • Send a 2-minute staff note: “No one should open unexpected ‘document’ files sent via WhatsApp. If it’s real, ask for it via email from a known address or via your approved portal.”
  • Block and report: if a suspicious WhatsApp message arrives, don’t forward it internally; take a screenshot, report to IT, and block the sender.
  • Check remote access tooling: confirm only approved remote support tools are installed and in use on company devices.
  • Review admin access: ensure MFA is enabled on email, Microsoft 365/Google, accounting, payroll, and any remote access services.
  • Quick finance control: remind staff that bank detail changes must be verified out-of-band (call a known number, not the one in the message).

Ask your IT provider

  • Do you have alerts for new/unauthorised RMM tools being installed on endpoints?
  • Can you confirm which remote support tools are approved for our business and how we spot anything else?
  • Are we monitoring for unusual logins (new country/device) to email and VPN/firewall accounts?
  • Do we have a documented process for rapid device isolation if someone clicks a malicious attachment?
  • If we use Fortinet/FortiGate (or a managed firewall), have you validated that our admin access and credentials are protected and being monitored?

Patch watch - only one short paragraph, and only if relevant

There is reporting of a long-standing issue in the Squid web proxy that could expose cleartext HTTP requests to other proxy users in some setups. This is mainly relevant if you run Squid internally (including via a supplier). Ask your IT provider whether you use Squid anywhere and, if so, what mitigations/updates have been applied.

One action today

Send a same-day staff reminder: don’t open any unexpected “invoice/PO/document” files sent via WhatsApp; verify the request using a known contact method and report the message to IT.
