Daily SMB Cyber Intelligence Brief

Today’s SMB cyber lookout: Microsoft 365 device‑code phishing, mobile banking malware, and router backdoor risk

What small and medium-sized businesses should look out for today.

High Tuesday 07 July 2026, 19:19 UK time
Today’s look-out: Account takeover scams (M365 device-code), banking fraud on mobiles, and small-office router compromise

What to look out for today

Three themes that are practical for SMEs to watch for today:

  • Microsoft 365 “device code” phishing using collaboration-style lures that push staff into a real Microsoft sign-in flow (no fake password page).
  • Android banking fraud being “rented” via Telegram, making it easier for criminals to run convincing mobile-led payment and account takeover scams.
  • Small office / home router risk: reports of a hidden backdoor in some Tenda router firmware that could grant admin access.

Why this matters to smaller businesses

  • M365 compromise can become an “everything compromise”: email, Teams, OneDrive/SharePoint access, password resets, invoice fraud, and supplier impersonation.
  • Mobile is now a direct path to money: if a staff member uses a personal or work Android phone for banking, payments approvals, or authentication codes, malware can intercept logins and one-time codes.
  • Routers are a single point of failure: a compromised router can enable eavesdropping, redirection, or access to internal systems—especially where remote access, CCTV, or card terminals share the network.

Warning signs

  • Emails/Teams messages saying “You’ve been invited to collaborate” or “urgent document access” followed by instructions to enter a code on a Microsoft sign-in page.
  • A colleague says they “didn’t enter a password, only a code”—and assumes it must be safe because it looked like Microsoft.
  • Unexpected MFA prompts, new device sign-ins, or inbox rules being created (e.g., auto-forwarding finance emails).
  • On Android: sudden accessibility permissions, “battery optimisation” prompts for unknown apps, overlays appearing over banking apps, or the phone becoming unusually slow/hot.
  • Router/admin panel behaving oddly: unknown admin accounts, settings changes, DNS changes, or remote management enabled without approval.

How attackers may exploit the situation

  • Device-code phishing: attacker persuades a user to enter a device code, granting the attacker an authenticated session to the user’s M365 account without stealing a password in the traditional way.
  • Mobile banking malware: criminals aim to steal bank credentials and capture one-time codes, enabling unauthorised payments and account takeover.
  • Router backdoor risk: if the router can be accessed with hidden/weak authentication, an attacker may gain admin control and tamper with network traffic or access exposed services.

What to do today

  • Brief staff (5 minutes): Microsoft sign-in screens can still be part of a scam. Don’t enter “device codes” unless they initiated the login on a trusted device and understand why.
  • Finance controls: require a second channel verification for new payee details and “urgent” payment requests—even if they come from a real colleague’s mailbox.
  • Check M365 basics: review recent sign-ins for finance and senior accounts; look for new inbox rules/forwarding; confirm MFA is enforced for all users.
  • Mobile hygiene: remind staff not to install unknown APKs or “support” apps from messages; review high-risk app permissions (Accessibility, Device Admin) on any phone used for work approvals.
  • Router sanity check: confirm what router model/firmware you use (especially if Tenda); ensure admin password is unique; disable remote admin unless explicitly needed; confirm DNS settings are expected.

Ask your IT provider

  • Can you block/limit device-code sign-ins where appropriate, and alert on unusual device-code authentication attempts?
  • Do we have conditional access or equivalent controls to reduce logins from unfamiliar locations/devices for finance/admin users?
  • What is our rapid response if an M365 account is suspected compromised (session revocation, password reset, MFA reset, mailbox rule cleanup, audit)?
  • Do we have an approved router list and a process to track firmware, admin access, and remote management settings for branch/home offices?
  • What’s our position on BYOD mobiles used for approvals/banking—do we need minimum security settings or MDM for those roles?

Patch watch - only one short paragraph, and only if relevant

If your business uses a Tenda router, treat today’s report as a prompt to identify the exact model/firmware in use and discuss mitigation with your IT support (including firmware updates or replacement). For most SMEs, the priority is confirming the device in use, locking down admin access, and disabling remote management unless essential.

One action today

Send a same-day staff note: “Never enter Microsoft ‘device codes’ from messages—only sign in when you initiated it; report any unexpected MFA prompts immediately.”
